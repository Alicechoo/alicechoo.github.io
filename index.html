之前磕磕绊绊终于装好了tensorflow，却不料毕设老师说你换个课题吧。于是猝不及防地开始了我的React-Native之路…
这篇文章主要是记录自己在运行react-native run-android遇到的一些问题，也是根据自己的回忆七七八八的凑起来的，有些地方也记得不太清楚了。官网上有相应的新手问答，囊括了各式各样的run-android可能遇到的问题。

安装React Native

在此之前，我没有装过相关的Android Studio以及JDK，所以全程按照官方文档一步一步操作，并没有遇到什么问题。中间Android Studio装的时间比较长（主要是Android SDK），不过我是放电脑下载就去睡觉了，第二天早上就装好了。所以搭建环境倒是很顺，直到运行了react-native run-android。

在电脑上运行没有报错后，就转去看手机。在真机上运行React Native可以参考官方文档中的在设备上运行，感觉可以少踩很多坑。如果你的设备是android 5.0以上版本(API 21+)，设备通过USB连接到电脑并打开USB调试模式后，运行

不需要更多配置，你就可以使用Reload JS和其它的开发选项了。
对于我自己的手机来说并没有跟我预想的一样出现Welcome to React Native界面，出现的只有白屏。等了一小会儿后，仍然是白屏。百度后发现是因为小米手机要开启悬浮窗权限，于是在手机桌面->安全中心->授权管理->应用权限管理->权限管理->显示悬浮窗->AwesomeProject设置允许显示悬浮窗，再点开AwesomeProject，此时出现了满眼的红屏。

按照红屏上的说明，摇一摇手机然后选择弹出框中的Dev Setting，然后点击Debug server host & port for device，设置与自己电脑相同的ip并且端口号设置为8081，ip可以在命令行运行ipconfig查看，如我的设置为：192.168.253.1:8081。

然后返回去摇晃手机，点击Reload。但是我的的界面还是红屏，不过此时的信息是不一样的

Windows10 64位使用Anaconda安装tensorflow

发表于 2017-10-31 | 分类于 安装tensorflow

本文使用Anaconda来安装tensorflow cpu版本，并希望在 Jupyter notebook 中导入 TensorFlow 。在安装tensorflow时，参照的是以前的安装教程，教程中的tensorflow当时还不支持Python3.6，因此在本文中都是按照tensorflow不支持Python3.6来进行。相信现在tensorflow支持Python3.6之后，安装tensorflow cpu版本会更加简便。

安装Anaconda

在Anaconda官网上选择windows Python3.6 64位下载，并安装。安装完成后，配置环境变量。

win10配置环境变量

windows10配置环境变量可以点击”设置”->”系统”->”关于”->右侧栏相关设置中的”系统信息”->左侧栏”高级系统设置”->”环境变量”来配置相应的环境变量。为用户变量中的Path添加相应Anaconda目录下的Scripts即可。例如我的Anaconda安装在D:\software目录下，此时将Path添加一条D:\Software\Anaconda\Scripts即可。

因为此时Anaconda默认支持python3.6，Anaconda中的Python，IPython，pip，Spyder，Jupyter notebook等都是Python3.6版本，此时若使用pip安装tensorflow会产生错误，因此我们要将Anaconda降级为Python3.5版本。
